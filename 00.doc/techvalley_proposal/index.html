0개요

본 문서는 테크밸리 산업용 X-ray/CT IoT·제조 통합 플랫폼의 전체 기능을 Phase 구분 없이 도메인 단위로 나열한 종합 기능정의서입니다. 장비 운영·정비를 담당하는 프로젝트 A(IoT 서비스 플랫폼)와 수주·생산·발주·재고를 담당하는 프로젝트 B(제조·ERP 플랫폼)로 구분하며, 제품주문(완제품 장비)과 부품주문(정비용 부품)을 분리 처리합니다. 모든 주문은 출하예정일·배송예정일 등 전 단계 일정과 서비스 수준(SLA)·서비스 가능 수준으로 관리됩니다.

AIoT 서비스 플랫폼 — 기능 목록

DAI 지원 플랫폼 — Bedrock + RAG (서비스 업무 향상)

서비스 엔지니어·상담 업무의 생산성을 높이기 위한 별도 AI 지원 영역입니다. Amazon Bedrock(Claude)과 RAG(정비 이력·매뉴얼 지식베이스)를 결합해 검색·요약·추천·진단 가이드를 제공합니다. 운영 IoT·ERP와는 API로 연계되는 독립 영역으로 구성합니다.

No기능 도메인세부 기능
D1정비 이력·매뉴얼 RAG 검색·요약정비 이력·장비 매뉴얼·기술문서 지식베이스(Knowledge Base) 구축 / 자연어 질의 → 근거 문서 검색·인용 / 핵심 요약 답변 생성
D2서비스 리포트·조치 자동 추천증상·알람·텔레메트리 입력 → 유사 사례 검색 / 권장 조치·부품·점검 항목 추천 / 서비스 리포트 초안 자동 작성
D3챗봇 기반 장애 진단 가이드대화형 진단 챗봇(엔지니어·상담사용) / 단계별 트러블슈팅 가이드 / 원격제어·서비스 호출·진행등록으로 핸드오프
D4지식베이스·파이프라인 운영문서 수집·청킹·임베딩 / 벡터 검색(OpenSearch 등) / 가드레일·인용 표기·환각 억제 / 피드백 기반 품질 개선

※ AI 지원 플랫폼은 운영(IoT)·제조(ERP)의 데이터를 읽어 보조 정보를 생성하며, 실제 제어·발주·정비 확정은 기존 IoT/ERP 기능이 수행합니다(사람 확인 기반).

⑨비고

    
    
      
  1. 전체 기능은 Phase 구분 없이 프로젝트 A(IoT)·B(제조·ERP)의 도메인 단위로 나열했습니다.
    2. 
      
  2. 제품주문(완제품 장비)과 부품주문(정비용 부품)은 발생 주체·트리거·전표·배송 형태·완료 기준이 달라 별도 도메인으로 처리합니다. 부품 주문(A10)과 AS(정비) 관리(A11)도 기능·화면·API를 분리합니다.
    3. 
      
  3. IoT 서비스 핵심 흐름은 알람 → 원격제어 → 서비스 호출 → 처리 진행 등록이며, 부품 주문·AS 완료는 이후 조건부·후속 단계로 연결됩니다.
    4. 
      
  4. 모든 주문은 주문일·생산예정·출하예정일·출하실적·배송예정일(ETA)·배송실적·설치/교체 완료일을 관리하며, 예정일 대비 지연 시 알림이 발생합니다.
    5. 
      
  5. 서비스 수준(SLA)은 Critical/High/Standard 3등급으로 응답·출동·복구·가동률 목표를 정의하고, 알람 시점 기준 자동 집계·에스컬레이션합니다.
    6. 
      
  6. 서비스 가능 수준은 부품 가용성·엔지니어 가용성·원격 해결 가능성·SLA 충족 가능성을 종합해 "즉시 원격가능 / 당일·익일 방문 / 부품 대기(예상 복구일)"로 실시간 표출합니다.
    7. 
      
  7. 연동은 동기 REST + 비동기 EventBridge(배송·예정일 이벤트 포함)이며, 마스터 기준은 장비/부품 = ERP, 운영 텔레메트리 = IoT입니다.
    8. 
      
  8. 데이터 마이그레이션은 Legacy MES → Aurora(DMS·Glue·CDC)를 중심으로 5단계로 수행하며 운영 중 정합을 유지합니다.
    9. 
      
  9. 인증·보안은 Cognito(웹)·X.509/IoT Policy(장비)·OAuth2/mTLS(연동)·Secrets Manager/KMS(자격증명)·IAM/RBAC(권한)·CloudTrail(감사)로 계층 분리 적용합니다.
    10.
